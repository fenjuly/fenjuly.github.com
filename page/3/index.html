说点什么吧

就要回家了，这学期就算真的结束了，回头想下这学期，感慨颇多啊.

梳理

从头开始想起我大概干的一些事：

  • 上学期结尾说过还是要回归书本，开学一两个月内都泡在图书馆内研究算法导论。算是没有做得太差。

  • 之后清明室友都回家了或者去旅游了，在寝室憋了三天把清水河畔的初版写完了，然后…其实觉得有点孤独。

  • 之后回到工作室，继续看东西学习，期间投了几个实习简历出去，反正才大二我更多的是混混面试经验。没有然后。

  • 之后和callofdutyops重写了清水河畔，其实就是改善了一下ui，没有什么多出实质性的功能

  • 后面和金振祖，茂哥去接了个餐饮管理系统的外包，河畔那边我就选择没有继续写了，在github上开源了，我们开了一个头，希望有人能够继续写下去。

  • 端午节的早上，唐为哥给我打了一个电话，叫我跟他们一家子出去玩。真的好高兴。 跟着他们去了国色天香，完了一个类似过山车的游乐设施，真吓人。晚上他们一家又接我去他们家吃饭，弄了龙虾，腊干子，等等。吃得我好high，这或许是我在这边读书感到最快乐的时候。

  • 后面就是主要花时间去做项目了，到了临近考试的那几天确实有点焦虑，因为没上过什么课，还好是熬过去了。然后，金振祖毕业回家了，他算是我为数不多的志同道合的人吧。25号早上和他一起拍了照片纪念纪念。然后茂哥也要走了，工作室集体请他和张哥吃了顿饭。最后走的是张哥，张哥算是我这个大学最敬佩的人了，教了我很多的东西，主要还是榜样作用。

  • 世界杯看得很high，有几个晚上每天都看到5点，然后12+起床。

  • 到了最近几天，很久前我大一上进工作室的时候就说要做工作室网站，结果一直没人做出来，呵呵。现在我接手工作室了，我还是决定一定要完成，就让这个坑在我这里填满。所以这几天一直在写工作室网站，进度不错，估计不要几天就可以挂上工作室服务器了。

  • 在学校的最后一天，大飞哥请了和工作室的其他几个人去他家吃饭，大飞哥手艺真不错，我吃了三碗饭+一瓶啤酒。

总结

  • 生活上感觉没有以前的焦虑了，做事也沉得下来了。其实下来就是改掉了不少坏习惯然状态更好了一点（比如打游戏，懒得洗衣服，懒得运动等等）。

  • coding上感觉视野也更广阔了，也懂的了要去弄懂一些原理性的动西而不是会用而已。

    总体上对自己的总结就是做得不够好，但是有进步，还有非常大的提升空间。

记个我忽略的问题

写这篇博客顺便整理一下思路
目前正在写的一个项目用到了mybatis，然后昨天遇到这么一个问题:

有两个Java类对应两张表，DishOrder.java(订单)和Dish.java(菜品).一个订单多个菜品
一个菜品属于一个订单.所以我们另外需要一个表(OrderRecord.java)订单记录,来记下菜品和订单间的关系.

我们查询一个用户的订单时,我们开始的查询语句和resultMap：

<select id="queryDishOrdersByUser" resultMap="DishOrderResultMap"
    parameterType="java.lang.Long">
    SELECT
    orderdish.send_time,
    orderdish.remark,
    orderdish.buyer_phone_num,
    orderdish.buyer_address,
    orderdish.order_num,
    orderdish.order_status,
    orderdish.order_date,
    dish.dish_name,
    dish.dis_price,
    dish.diff,
    dish.diff_name,
    dish.diff_price,
    orderrecord.dish_copies,
    dish.dish_picture,
    restaurant.username AS res_name,
    orderdish.res_id AS res_id,
    orderrecord.dis_id AS dis_id,
    orderdish.use_id AS user_id,
    orderdish.id AS orderdish_id,
    orderrecord.ord_id,
    orderrecord.id AS orderrecord_id
    FROM
    orderdish
    INNER JOIN orderrecord ON orderrecord.ord_id = orderdish.id
    INNER JOIN dish ON orderrecord.dis_id = dish.id
    INNER JOIN restaurant ON orderdish.res_id = restaurant.id
    WHERE
    orderdish.use_id = #{id}
</select>

            <result property="diff" column="diff"/>
            <result property="diffName" column="diff_name"/>
            <result property="diffPrice" column="diff_price"/>
        </association>
</collection>
</resultMap>

然后我的数据库里有三个订单,第一和第二个订单分别都只包含一个菜品,故只存在一个OrderRecord,第三个订单有两个菜品信息,故有两个OrderRecord,理想的查询结果应该是这样的.简约的写下:

{
    "id":1,
    "orderrecords":[
    {"id":1, "dish_copies":1, "dish_id":1, "orderrecord_id":1}
    ]
}
{
    "id":2,

    {"id":4, "dish_copies":2, "dish_id":2, "orderrecord_id":3}
    ]
}

而实际上按照我开始的resultMap返回的结果是

{
    "id":1,
    "orderrecords":[
    {"id":1, "dish_copies":1, "dish_id":1, "orderrecord_id":1}
    ]
}
{

    ]
}
{
    "id":3,

    {"id":3, "dish_copies":1, "dish_id":1, "orderrecord_id":3}
    ]
}
{
    "id":3,
    "orderrecords":[
    {"id":4, "dish_copies":2, "dish_id":2, "orderrecord_id":3}
    ]
}

同一个订单的订单记录并没有合在一起…

最后找到的原因就是reusltMap应该这么写:

<resultMap type="org.xhome.cy.model.DishOrder" id="DishOrderResultMap" extends="BaseResultMap" >
<id property="id" column="orderdish_id"/>
<association property="restaurant" column="res_id" javaType="Restaurant">
    <result property="username" column="res_name"/>
</association>
<collection property="orderRecords" javaType="list" ofType="OrderRecord" column="orderdish_id">
           <id property="id" column="orderrecord_id"/>
        <result property="dishCopies" column="dish_copies" />
        <association property="dish" column="orderrecord_id" javaType="Dish">
            <id property="id" column="dis_id"/>
            <result property="dishName" column="dish_name"/>
            <result property="disPrice" column="dis_price"/>
            <result property="dishPicture" column="dish_picture"/>
            <result property="diff" column="diff"/>
            <result property="diffName" column="diff_name"/>
            <result property="diffPrice" column="diff_price"/>
        </association>
</collection>
</resultMap>

细心的同学发现我加上了id,这个属性,然后在association里面column的值为这个id.

最后总结下来大概就是这么回事: sql语句查找出来确实是四条结果，但是有两个结果是需要合并的,mybatis就是通过
id这个属性来判断该不该合并…

记一个很诡异的问题

前段时间做android用到了Picasso这个图片下载缓存库，确实用着很方便，但有个很隐晦的问题困扰了我好久。

看Picasso的官网，很容易发现Picasso的一般用法是这样的: 

Picasso.with(context).load("http://i.imgur.com/DvpvklR.png").into(imageView);

但是我们注意,如果你要对下载下来的图片进行处理，比如大小按比例裁剪等等.那么你就要这样用:

Picasso.with(context).load("http://i.imgur.com/DvpvklR.png")
                        .into(new Target() {

                            @Override
                            public void onPrepareLoad(Drawable arg0) {

                            }

                            @Override
                            public void onBitmapLoaded(Bitmap bitmap, LoadedFrom loadedFrom) {
                                //do something here
                            }

                            @Override
                            public void onBitmapFailed(Drawable arg0) {

                            }
                        });

在OnBitmapLoaded得到bitmap，然后进行处理. 说道这里大家都以为没有问题了，但是你会发现
当你手机下载一张图之后本应该要显示的却没有，然而你拖动屏幕之后就有显示了.这里如果你调试时你会发现，第一次这个OnBitmapLoadeded根本没有调用.

这个问题研究了好久，最后在https://github.com/square/picasso/issues/251上找到了答案

Your CustomTarget instance gets gc’ed. You need to hold a strong reference to it.

这就是答案 -.-|. 被gc回收掉了，所以我们必须要加强引用

You need to hold a strong reference to targets yourself.

CustomTarget target = new CustomTarget(imageView, progressBar,
        index);
    picasso.load(imageUrl).into(target);

after this line target has nothing referencing it so it will be garbaged collected.

这是这个人最后的解决方案:

public class ImageLoadWithCallback extends FragmentActivity {

    String[] imageUrls = {
            "http://digital-photography-school.com/wp-content/uploads/2013/03/Acorn256.png",
            "http://stereo-ssc.nascom.nasa.gov/beacon/latest_256/behind_euvi_195_latest.jpg",
            "http://littleblackapp.blob.core.windows.net/product3071/img10020_0.png" };

    private Picasso picasso;
    private ArrayList<CustomTarget> targetList;


    @Override
    protected void onCreate(Bundle arg0) {
        super.onCreate(arg0);
        setContentView(R.layout.activity_main);
        picasso = Picasso.with(this);
        initLoadImages();
    }

    private void initLoadImages() {
        // for holding strong reference to Target so that it won't be GC'ed
        targetList = new ArrayList<ImageLoadWithCallback.CustomTarget>(3);
        for (int i = 0; i < imageUrls.length; i++) {
            String imageUrl = imageUrls[i];
            picasso.getSnapshot().dump();
            CustomTarget target = new CustomTarget(imageView, progressBar, i);
            targetList.add(i, target);
            picasso.load(imageUrl).into(target);
        }

    }

    @Override
    protected void onDestroy() {
        super.onDestroy();
        if (isFinishing()) {
            for (CustomTarget target : targetList)
                picasso.cancelRequest(target);

            targetList.clear();
            targetList = null;
        }

    }

    public class CustomTarget implements Target {
        public ImageView imageView;
        public ProgressBar progressBar;
        private int index = -1;

        public CustomTarget(ImageView imageView, ProgressBar progressBar,
                int position) {
            this.imageView = imageView;
            this.progressBar = progressBar;
                      this.index =position;
        }

        @Override
        public void onBitmapLoaded(final Bitmap bitmap, LoadedFrom from) {
            imageView.setImageBitmap(bitmap);
            releaseTargetReference();
        }

        @Override
        public void onBitmapFailed() {
            releaseTargetReference();
        }

        private void releaseTargetReference() {
            if (index != -1)
                targetList.remove(index);
        }

    }

}
