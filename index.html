又来了，这次是一个很Cool的Button

先上图：

起因是前几天我在逛dribbble的时候看见了这个https://dribbble.com/shots/2012292-Download-Animation，觉得awesome！，就动了自己把它实现的念头。

思路

整体来看有点复杂，不过可以把实现过程按照阶段来划分。

先下载原图，观察每一帧。

第一阶段

就是箭头的柄慢慢压缩，箭头的头慢慢伸展，最后柄合成一个小球，被箭头弹起来。设如图a,b,c,d,e五个点，其中a,e点一开始是重合的。假设在实际绘制中，把这一阶段设为18帧，那么前9帧，先绘制a,b两点慢慢靠近（即a.y-step a.b+step。c,d,e三点慢慢舒张（即，每隔3帧c向左移，d向右移，e向下移，c.x-step b.x+step, e.y + step）。10-12帧，c,d,e连城弧线向上弹，同时小球弹飞（c.x-step c.y-step d.x+step d.y-step e.y-step jump.y - step）。13-16帧，c,d,e弧线慢慢变为水平线，小球落下（jump.y-step）。

第二阶段：

这一阶段由两个动画组成，最外层的progress动画，和圆心水平线的波浪动画。最外层的progress动画比较简单，就是每隔一帧按照当前的progress来绘制圆弧
，圆弧的角度为progress/100 * 360。圆心的波浪动画采用三角函数来做，我这里的做法是取了两个周期，

然后每隔1/8个周期选一个点，那么就有17个点。假设每一帧我是隔20ms刷新的，那么我就可以将周期设为160ms。那么这里我就可以求出每一个点的高度随时间变化的函数：
y = H * sin(Pi/80 * (originalTime + currentTime))。（这里有个小细节，H是随progress的进度变化而变化的）。这里求出每点的高度之后，我在绘制每一帧的时候就可以用连线将各个点连接起来，然后随着帧数的增加，currentTiem也增加，currentTiem=curremtTime+20ms。这样就形成了波浪效果。

第三阶段：

这一阶段比较简单了，就是波浪线消失，然后慢慢出现小勾。假设这一阶段我定为4帧，那么前3帧（c.x+step c.y+step e.y+step d.x-step d.y-step），第4帧小勾“抖动放大”（c.x-step c.y-step e.y+step d.x+step d.y-step)。

注：以上的step值都不一定相同，只是用step来代替每次一次的变化量。

好嘞，以上就是实现过程。
然后附上源码地址：GitHub :)

这次是一个Loader

前几天在食堂排队时刷tweet，然后见看到了这个东西http://t.co/sUuYTeKOzp。觉得十分有趣，就想着去实现一下，于是就有了这篇文章。先看效果图。

思路

整体来看有八个固定的点，一个动态的点，然后整体效果是由三个动画合成的：1.动态点旋转动画。2.动态点与固定点接触时，固定点半径的放大和缩小动画。3.动态点与固定点接触与分解时的”粘连“动画。

做法

  • 首先定义几个东西
    • 大圆的圆心(bigCircleCneterX, bigCircleCenterY)。
    • 大圆的半径raduis。
    • 固定点的半径circleRaduis。
    • 动态点的半径moveRaduis。
    • 动态点每次移动的角度STEP。
    • 固定点半径的增益长度ADDITION_LENGTH。
    • 动态点的圆心与固定点圆心的距离distance。动态点是否与固定点相交：
  • 然后画八个固定点，只需要确定圆心和半径即可
  • 动态点的绘制

  • 固定点与动态点相交时的半径变化
    当相交时半径开始变化，二者的圆心离得越近固定点的半径增益越大。利用前面提到的判断是否相交的函数，如果相交则半径增益的计算规则为：
    srealRaduis = circleRaduis + (1 - distance / (circleRaduis + moveRaduis))。

  • 动态点与固定点接触与分解时的”粘连“动画
    这个效果与手Q 的红点消除效果差不多，都是用贝塞尔曲线做的。
    盗用两张图（图片来自http://isux.tencent.com/qq-mobile-off-duty.html）：

要做出“粘连”的效果，关键是要计算出如图的p1 p2 p3 p4还有P5这几个点。p1-p4是两个圆的切点，p5这个点是贝塞尔曲线的控制点，这个里面我取的是固定点与动态点圆心连线的中点。具体怎么计算出p1-p4并绘制，我参考了这个项目https://github.com/chenupt/SpringIndicator

Sketch

Medium

Else

开学一个月的总结

开学这一个月别的事没干，主要就是准备实习的事。基本上就是看面试题，然后投简历，然后面试。到今天为止，投了阿里，百度，腾讯，豆瓣，网易，知乎，豌豆荚。

从时间顺序来：

阿里

之前寒假在家投了阿里的内推，然后开学第一周就收到了阿里的电面。之前投职位的时候选择了研发工程师这个职位，并没有找到我想投的Android开发，然后面试的时候就出问题了。面试官是做web的一上来就问我“你怎么理解JVM？”。听到这个问题我心里就开始担心了，因为自己主要精力都放在了Android这一块，然后Java Web这一块就没有好好研究。过程就不说了，结果就是挂了。然后最近又投了阿里的正式校园实习生招聘。

网易

阿里电面后看了一个星期左右的面试题。然后某天在V2EX上看见了网易的邮箱大师部门招实习生，看见有Android职位，我就立即投了简历。网易的效率真的高，第二天就中午就打电话过来约面试时间，商量了一下定在了下午四点面试。下午四点面试开始。
然后下面是大概问的问题：

  • AsyncTask内部实现，为什么可以在onPostExcute方法里面更新UI。
  • Handler，Message，Lopper。
  • HTTP协议的一些基础知识，GET 和 POST有什么区别。
  • 网络请求你是怎么做的？
  • 问了我简历上的一些项目。

第一个问题我说的很模糊，具体我本来也不是很清楚，我就说是用线程池实现的，之所以能在onPostExcute方法里曼更新UI是因为同时也封装了handler。然后网络请求我就说我用的google的Volley这个库。然后我就把这个库的用法，怎么实现一个网络请求，怎么自定义一个Request说了一遍。

感觉还不错，还有一些别的问题基本上都答上了来了，然后那边就直接约我二面了，不过一直到现在我都没有收到二面的消息。

豆瓣

面网易的那个下午2点我还参加了豆瓣的在线笔试，我选的Android方向，一个小时12道题目，2道算法题任选一道做，7道Android基础题，3道android进阶题。

2道算法题算是非常简单了，第一道是求一个数组里面第k大的数，我觉得就是变相考你的快排。第二道是一个数组里面有正数和负数，求子数组和的最大值，也不是很难。

7个基础题无非就是Activity的生命周期，Android四大组件的一些知识

3个进阶题：

  • 什么是OOM，怎么解决？
  • 发现ListView卡顿，有什么方法优化？

  • 设计本地网络图片缓存策略。

    前两个没什么问题，最后一个不是很清楚，乱答的。。。然后笔试就完了，然后到现在也没有收到面试通知。

知乎

之后就投了知乎的简历，然后定在很近的一个周末进行面试。总的来说第一面的话基本上也是考一些Android的基础知识，不过有一个地方我没有答上来，就是Android间的进程通信，我回答的是通过Intent进行通信的，不过这是单方面的，然后面试官告诉我另外一个是aidl。面试完后我去查了查，虽然概念不太好理解，但是用起来却是超级简单！最好的一个例子就是播放音乐时，播放界面的Activity和播放音乐的Service之间的通信。完了后，面试官也约了二面的时间。

二面是接下来一周的周一晚7点开始的。这一面的话就问了一个ListView的item重用问题，然后就没有问Android相关的问题了。面试官一开始问Java中的Object类有哪些方法，然后问了hashcode方法有什么用，equals方法和”==”的区别，notify和wait方法有什么用，同步和异步的区别等一些问题。然后总的答下来还觉得ok。然后面试官问了一个小问题：两个单向链表怎么判断他们是否相交？ 这里我傻逼了，明明是单向链表，我死死地在考虑交叉的问题。还好没什么大碍～

接下来的三面hr跟我说是知乎的CTO李申申来面我，听到这个还是吓到了，准备三面的前两天心里还是有点紧张的。不过幸运的是三面的时候申哥并没有问我一点技术问题，而是问了我的日常生活状态，每天干什么事，喜不喜欢读书，遇到问题时是怎么解决的，最后问了我对知乎这个产品，这个团队的看法。我还是认真地说了下我的看法，总的下来还不错吧。然后知乎的offer算是拿到了。

总结

这一个月还是收获非常多！首先是面试题中学到了很多以前忽略的点，然后就是收获了非常宝贵的面试经验。这里先记录下这个月的事迹，接下来还有几个公司的笔试和面试需要准备，还是加油吧～
